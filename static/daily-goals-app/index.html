What you'll build

You'll build an app that lets people use their phone to create and update lists of their goals for the day.

It'll have the following features:

  • A heading with the name of the app
  • A heading with the person's name
  • A heading for the person's goals
  • A list of their goals
  • Headings that look good
  • A list that looks good
  • The person can update and save their name
  • The person can update and save their goals

Here's what it might look like:

What you'll learn

  • How to write HTML code.
  • How to style HTML code with CSS classes.
  • How to make an app out of HTML and CSS.

What you'll need

  • A login to Repl.it
  • That's it!
  1. Login to Repl.it and create a new HTML, CSS, and JS repl.

  1. Delete the script.js and style.css files (we won't be using these)
  2. Delete all the starting code in index.html (we'll be writing our own)
  3. Rename your repl to something more appropriate, such as daily-goals

Now you're ready to get coding!

Try out some HTML code

Add this code to your index.html and click Run (green play button at the top of the page)

<h1>
  Daily Goals App
</h1>

You'll see "Daily Goals App" written in big bold letters.

<h1> stands for heading 1, which is the biggest heading. It shows where the heading starts.

You'll notice there's a </h1> with a slash at the start. This shows where the heading ends.

<h1> is an opening tag, and </h1> is a closing tag. Together, they make an HTML element.

Try using some of these tags on your page.

  • <h2> - heading 2, which is slightly smaller than heading 1
  • <h3> - I think you get the idea
  • <p> - paragraph, which is just some normal text

This should be a heading 2 <h2> because it's almost as important.

This should be a heading 2 <h2> because it's as important as the name.

Add the HTML code for the list

Each goal is an item in a list, or a list item.

The HTML tag for a list item is <li>.

Try adding some goals as list items to your page, under the goals heading.

<h2>
  Daily Goals
</h2>
<li>Learn HTML</li>
<li>Try out list items</li>

Usually, list items will be inside a list element.

The HTML tag for a list is <ul> which stands for unordered list. Unordered lists have bullet points.

<ul>
  <li>Learn HTML</li>
  <li>Try out list items</li>
</ul

Alternatively, you could use <ol> which stands for ordered list. Ordered lists are numbered.

<ol>
  <li>Learn HTML</li>
  <li>Try out list items</li>
</ol

You decide which type of list you prefer.

UI stands for user interface, which is the web page and its design.

You can add CSS classes to your HTML code to change the visual appearance and improve the UI.

The classes you'll add are from a website called Fomantic UI. The contributors there have written lots of CSS code that you can use for free.

Connect your project to Fomantic UI by adding this line at the very bottom of index.html

<script src="https://rrgs.nz/scripts"></script>

Style the main heading

The header class makes the heading big and bold.

This helps the user see what information is where on the page.

You can also add the center aligned class to put the header in the middle.

Fomantic page on headers

    <h1 class="ui center aligned header">
      My First Web App
    </h1>

Make the body a container for everything else

The container class adds some space around the thing it's added to.

This makes the page look less cluttered.

Fomantic page on containers

  <body class="ui container">

Make the sections containers for the inputs and list

You'll re-use the container class for the sections.

This again helps spread things out and make the page look less cluttered.

Fomantic page on containers

    <section class="ui container">
      <input>
    </section>
    <section class="ui container">
      <ul>
        <li></li>
      </ul>
    </section>

Mavo is the thing that will make your web app interactive for the user, letting them input their name and goals.

The first thing you need to do is define which parts of your HTML code should be a Mavo app (mv-app)

Mavo page on defining a Mavo app

My name

The My name section will need to be edited and saved.

You'll call this section of the app name.

    <h2 class="ui block header">
      My name
    </h2>
    <section class="ui container"  mv-app="name">
      <input>
    </section>

Daily goals

The Daily goals section will also need to be edited and saved.

You'll call this section of the app goals.

    <h2 class="ui block header">
      Daily goals
    </h2>
    <section class="ui container" mv-app="goals">
      <ul>
        <li></li>
      </ul>
    </section>

Mavo lets you store the user's information.

This lets the app remember the user's name and goals if they leave and come back.

Mavo page on defining the storage

Both the name app and goals app information need to be saved.

The type of storage is local.

This means the information is only stored on the user's computer. It is never sent over the Internet.

    <h2 class="ui block header">
      My name
    </h2>
    <section class="ui container"  mv-app="name" mv-storage="local">
      <input>
    </section>
    <h2 class="ui block header">
      Daily goals
    </h2>
    <section class="ui container" mv-app="goals" mv-storage="local">
      <ul>
        <li></li>
      </ul>
    </section>

Mavo needs to know which parts of the HTML code need to be saved.

You'll do this by adding the property attribute to the relevant HTML.

My name

The input needs to be saved by Mavo.

You'll give it the property of name.

      <input property="name">

Daily goals

Each list item - which is one of the user's goals - needs to be saved by Mavo.

You'll give it the property of goal.

You'll add mv-multiple so the user can add more than one goal.

      <ul>
        <li property="goal" mv-multiple></li>
      </ul>

Your app is complete.

The user can add their name and goals, hit save, and your app will remember them for the user's next visit.
