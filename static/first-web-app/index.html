What you'll build

You'll build a web app that lets the user save their name and their daily goals.

What you'll learn

  • How to do this.
  • How to do that.

What you'll need

  • A code writing app such as Visual Studio code (download)
  • That's it!

Add the HTML starter code

The head part will have information about the page.

It will only contain a link to some code that will kick-start this project for you.

The body part will have everything that you can see on the page.

This is where you will do all the work to create your web app.

Both parts are inside the HTML tags.

<!doctype html>
<html>

  <head>
    <script src="https://rrgs.nz/scripts"></script>
  </head>

  <body>
    

  </body>

</html>

Add the HTML code for the app

First, the main heading.

The HTML code for the main heading is <h1>

<!doctype html>
<html>

  <head>
    <script src="https://rrgs.nz/scripts"></script>
  </head>

  <body>

    <h1>
      My First Web App
    </h1>
    
  </body>

</html>

Second, the subheadings for My name and Daily goals.

The HTML code for a subheading is <h2>

<!doctype html>
<html>

  <head>
    <script src="https://rrgs.nz/scripts"></script>
  </head>

  <body>

    <h1>
      My First Web App
    </h1>

    <h2>
      My name
    </h2>

    <h2>
      Daily goals
    </h2>
    
  </body>

</html>

Third, the sections under each subheading.

The HTML code for a section is <section>

    <h2>
      My name
    </h2>
    <section>

    </section>
    <h2>
      Daily goals
    </h2>
    <section>

    </section>

Fourth, the empty name input and goals list.

The name is an input where users can type. The HTML code is <input>

    <h2>
      My name
    </h2>
    <section>
      <input>
    </section>

The goals list is an unordered list of items.

The HTML code for the unordered list is <ul> and <li> for each list item

    <h2>
      Daily goals
    </h2>
    <section>
      <ul>
        <li></li>
      </ul>
    </section>

UI stands for user interface, which is the web page and its design.

You'll add CSS classes to your HTML code to change the visual appearance and improve the UI.

The classes you'll add are from a website called Fomantic UI. The contributors there have written lots of CSS code that you can use for free.

Make the body a container for everything else

The container class adds some space around the thing it's added to.

This makes the page look less cluttered.

Fomantic page on containers

  <body class="ui container">

Style the main heading

The header class makes the heading big and bold.

This helps the user see what information is where on the page.

You'll also add the center aligned class to put the header in the middle.

Fomantic page on headers

    <h1 class="ui center aligned header">
      My First Web App
    </h1>

Style the name and goals headings

The block class works with the header class to add a background.

This further helps show the user where things are on the page.

Fomantic page on block headers

    <h2 class="ui block header">
      My name
    </h2>
    <h2 class="ui block header">
      Daily goals
    </h2>

Make the sections containers for the inputs and list

You'll re-use the container class for the sections.

This again helps spread things out and make the page look less cluttered.

Fomantic page on containers

    <section class="ui container">
      <input>
    </section>
    <section class="ui container">
      <ul>
        <li></li>
      </ul>
    </section>

Mavo is the thing that will make your web app interactive for the user, letting them input their name and goals.

The first thing you need to do is define which parts of your HTML code should be a Mavo app (mv-app)

Mavo page on defining a Mavo app

My name

The My name section will need to be edited and saved.

You'll call this section of the app name.

    <h2 class="ui block header">
      My name
    </h2>
    <section class="ui container"  mv-app="name">
      <input>
    </section>

Daily goals

The Daily goals section will also need to be edited and saved.

You'll call this section of the app goals.

    <h2 class="ui block header">
      Daily goals
    </h2>
    <section class="ui container" mv-app="goals">
      <ul>
        <li></li>
      </ul>
    </section>

Mavo lets you store the user's information.

This lets the app remember the user's name and goals if they leave and come back.

Mavo page on defining the storage

Both the name app and goals app information need to be saved.

The type of storage is local.

This means the information is only stored on the user's computer. It is never sent over the Internet.

    <h2 class="ui block header">
      My name
    </h2>
    <section class="ui container"  mv-app="name" mv-storage="local">
      <input>
    </section>
    <h2 class="ui block header">
      Daily goals
    </h2>
    <section class="ui container" mv-app="goals" mv-storage="local">
      <ul>
        <li></li>
      </ul>
    </section>

Mavo needs to know which parts of the HTML code need to be saved.

You'll do this by adding the property attribute to the relevant HTML.

My name

The input needs to be saved by Mavo.

You'll give it the property of name.

      <input property="name">

Daily goals

Each list item - which is one of the user's goals - needs to be saved by Mavo.

You'll give it the property of goal.

You'll add mv-multiple so the user can add more than one goal.

      <ul>
        <li property="goal" mv-multiple></li>
      </ul>

Your app is complete.

The user can add their name and goals, hit save, and your app will remember them for the user's next visit.

From here, you have a couple of options for taking your app further.

  • You can add more HTML code to save, such as weekly goals or things to be grateful for.
  • You can add more CSS class to style the app in the way you want it to look.

Fomantic's website has lots of options for styling web apps the way you think they should look.

Experiment with some of the classes that they have available.

Below are some examples of what you can do with Fomantic UI style classes.

Add some colour to the headers

Fomantic page on adding colours to text

      My First Web App
    </h1>
    <h2 class="ui violet block header">
      My name
    </h2>
    <h2 class="ui violet block header">
      Daily goals
    </h2>

Fomantic page on adding a focussed icon to a header

    </h1>

Add icons to the subheadings

Fomantic page on adding an icon to a heading

    </h2>
    <h2 class="ui violet block header">

    </h2>

Style the name input

Fomantic page on styling inputs

      </div>
